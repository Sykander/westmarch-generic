## [1.1.0](https://github.com/Sykander/westmarch-generic/releases/tag/1.1.0)

> Planned onboarding release for `westmarch-generic`: make the first subscribed-server experience friendlier while preserving `westmarch_config` as the server-owned configuration handoff.

### Upgrade Steps

These steps are for server owners, config maintainers, and developers adopting the release.

- No breaking migration is planned for already configured servers.
- Servers without `westmarch_config` should treat any starter-world behavior as a temporary trial state until they create and wire their own config gvar.
- After creating or publishing a config gvar, set `!svar westmarch_config <your-gvar-id>` and rerun `!westmarch show`.

### Breaking Changes

- None planned.

### New Features

- Planned: normal commands can use the Forgotten Realms 2014 starter config when `westmarch_config` is unset.
- Planned: `!westmarch setup` and `!westmarch show` clearly report when no `westmarch_config` svar has been set.
- Planned: editor first-config flow reminds configurators to set `!svar westmarch_config <gvar-id>` after publish or export.
- Planned: public configurer guide library under `docs/guides/`, starting with a glossary and focused guides for server settings, display, policies, world data, encounters, encounter templates, publishing, validation, troubleshooting, and launch checks.
- Planned: companion-bot setup guides for common first-server tools such as Dyno, Tupperbox, and Bard Bot.
- Planned: expose `docs/guides/` through the GitHub Pages surface under `/westmarch-generic/docs/guides/`.
- Under investigation: a Forgotten Realms starter quest that teaches travel, location, economy, exploration, gathering, and quest commands through a Waterdeep to Silverymoon journey.
