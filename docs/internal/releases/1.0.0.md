# Release 1.0.0

Status: planned.

`1.0.0` is the first full public release of `westmarch-generic`. It should include the current implemented feature set from the pre-public `0.x` line, plus enough docs, validation, tests, and release hardening that server owners can adopt it without reading internal planning notes.

Factions are not part of `1.0.0`.

## Release promise

The `1.0.0` release should provide a generic, config-owned westmarch engine:

- server-specific behavior comes from svars and config gvars;
- commands are gated by `subsystems`;
- configs can be checked in the web editor;
- server owners can start from documented presets;
- no server-specific setting data is hard-coded into aliases.

## Included command surface

The public baseline includes the command families already present in `src/aliases`:

| Area | Commands |
|------|----------|
| Admin/setup | `westmarch`, `westmarch setup`, `westmarch show` |
| Exploration | `enc`, `forage`, `fish`, `mine`, `lumber`, `hunt`, `loot` |
| Travel | `travel`, `location`, `time`, `weather` |
| Downtime | `downtime` |
| Crafting | `craft`, `brew`, `enchant`, `scribe` |
| Economy | `job`, `buy`, `sell`, `wallet` |
| Content | `library`, `read` |
| Misc | `quest`, `recipe` |

Command availability still depends on server config. A command being part of `1.0.0` does not mean it is enabled in every preset.

## Included engine and config work

`1.0.0` should include:

- config loading through `westmarch_config` svar and config gvar ids;
- display, auth, policies, rules-version, and command-toggle helpers;
- world data for locations, paths, journeys, transport, calendars, weather, and biomes;
- generated catalogues for monsters, items, spells, books, and recipes;
- economy helpers for shops, stock, currencies, buy/sell, wallet, and jobs;
- crafting and downtime helpers;
- content/library helpers;
- misc quest and recipe helpers;
- dev/prod sourcemaps and generated env gvars from the approved workshop UUID set.

## Included editor work

`1.0.0` should include the web config editor as the recommended validation surface:

- load or paste config source;
- structured editing for implemented config areas;
- Check page with errors, warnings, field links, and suggested fixes;
- export for manual publishing;
- browser-side publish only if the current implementation and Avrae API behavior are verified for release.

## Included presets and docs

`1.0.0` should include:

- public setup guide in `docs/setup.md`;
- public documentation overview in `docs/README.md`;
- admin setup/show help that matches the implemented runtime;
- generic starter config;
- Forgotten Realms starter config at whatever scope is completed and validated before the tag.

The Forgotten Realms starter should be truthful about scope. A Sword Coast starter is acceptable; claiming complete coverage of all Forgotten Realms is not.

## Release blockers

Before tagging `1.0.0`:

- `package.json` version is updated to `1.0.0`;
- `make build` succeeds;
- `make test` succeeds;
- editor tests and build succeed;
- sourcemap dev/prod checks succeed;
- generated files are updated only through the build pipeline;
- docs do not describe implemented commands as planned stubs;
- the manual initial-release checklist has no unresolved release-blocking issues;
- public setup docs explain how to configure, validate, and deploy a server config.

## Explicitly out of scope

- Factions subsystem and `!faction` command.
- Discord role synchronization.
- Breaking schema migrations after the public tag.
- Full coverage of every Forgotten Realms region.

Post-release additive features belong in `1.x.x`, not `1.0.0`.
