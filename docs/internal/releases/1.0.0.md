## [1.0.0](https://github.com/Sykander/westmarch-generic/releases/tag/1.0.0)

> First public release of `westmarch-generic`: a configurable Avrae westmarch engine where server-specific behavior lives in svars and config gvars, commands are gated by `subsystems`, and server owners validate/edit configs through the web editor instead of forking alias code.

### Upgrade Steps

These steps are for server owners, config maintainers, and developers adopting the release.

- [ACTION REQUIRED] Create or load your server config in the web config editor and fix all blocking Check page errors before setting `westmarch_config`.
- [ACTION REQUIRED] Servers enabling `time` must define `world_data.calendars`; servers enabling `weather` must define `world_data.weather.by_area`.
- [ACTION REQUIRED] Servers enabling `buy`, `sell`, or `wallet` should define top-level `shops` and/or `currencies` as needed. `sell` requires at least one visible shop with `accepts_sells: True`.
- For larger maps, move locations and paths into external JSON gvars and reference them with `world_data.locations_gvar_id` and `world_data.paths_gvar_id`.
- Use the editor Check page as the config validation surface. `westmarch check` is retired; `westmarch show` remains the Discord-side config summary.

### Breaking Changes

- None for public users; this is the first public release.
- `1.0.0` establishes the backwards-compatibility baseline for documented commands, argument layouts, and config keys. Breaking contract changes after this release should wait for `2.0.0`.
- The first public config contract is that server-specific data belongs in config gvars loaded through the `westmarch_config` svar, not hard-coded into generic aliases.

### New Features

- Added the public command baseline:
  `westmarch`, `westmarch setup`, `westmarch show`, `enc`, `forage`, `fish`, `mine`, `lumber`, `hunt`, `loot`, `travel`, `location`, `time`, `weather`, `downtime`, `craft`, `brew`, `enchant`, `scribe`, `job`, `buy`, `sell`, `wallet`, `library`, `read`, `quest`, and `recipe`.
- Added config loading through the `westmarch_config` svar, with merged runtime defaults for display, auth, policies, rules version, subsystems, and command toggles.
- Added world-data support for locations, paths, journeys, transport modes, transport icons, calendars, weather areas, biome registries, and external location/path gvars.
- Added travel route planning with route priorities, transport requirements, tracked journey progression, and optional arrival time/weather/shop output.
- Added generated catalogues for monsters, items, spells, Forgotten Realms books, and recipes.
- Added economy support for configured shops, stock, services, buy/sell flows, wallet currencies, and jobs.
- Added crafting and downtime support for mundane crafting, brewing, enchanting, scribing, resource policies, recipe modes, item output modes, and tracked/manual downtime.
- Added content and misc support for libraries, reading, quests, and recipes.
- Added the static web config editor with starter loading, raw source editing, guided config controls, related-gvar loading, validation, export, and guarded browser-side publish.
- Added starter config sources for the minimal starter, Forgotten Realms 2014, and Westmarch 2014.
- Added dev/prod sourcemaps and generated env gvars from allocated workshop UUIDs.

### Bug Fixes

- Replaced placeholder source-adjacent alias help docs with player-facing help.
- Fixed missing exploration-pool messages so inferred biomes include location context.
- Implemented `time` and `weather` instead of exposing them as planned stubs.
- Improved buy/sell diagnostics for missing shops, wrong location, no sell support, empty stock, and query misses.
- Fixed travel path rendering so blank encounter-step biomes do not produce malformed `Run !enc ` output.
- Added editor validation for incomplete path encounter steps when location inference is not configured.
- Added `scribe -i` support to bypass only known-spell and spell-slot checks for feature-granted spells.
- Limited cooldown controls and warnings to commands that actually enforce runtime cooldowns.
- Added editor warnings/errors for deferred policy flags and dependency-sensitive options.
- Added tests to keep editor starter sources aligned with canonical starter config files.

### Performance Improvements

- Sharded generated monster, item, spell, book, recipe, biome, location, and path data so runtime lookups can load narrower data sets.
- Added hot-path catalogue search coverage for common monster lookup families.
- Added statement-budget coverage for large Forgotten Realms library and travel/pathfinding scenarios.
- Supported origin-indexed path gvars through `paths_by_from` so large route graphs can be read by origin instead of as one flat map.

### Other Changes

- Documented the first public setup flow in `docs/setup.md` and the overview/config model in `docs/README.md`.
- Added a pre-tag command contract review in `docs/internal/research/1.0.0-command-contract-review.md`.
- Documented next-major roadmap items in `docs/internal/releases/2.0.0.md`, including factions, combat handoff/maps, and economy progression commands.
- Deferred factions, Discord role synchronization, first-class combat map handoff, economy progression commands, breaking schema migrations, and full Forgotten Realms regional coverage to future work tracked from the `2.0.0` roadmap.
