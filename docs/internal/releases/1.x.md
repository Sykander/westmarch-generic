# Release 1.x.x roadmap

Status: planned.

The `1.x.x` line is for additive features after the `1.0.0` public baseline. These releases should preserve existing `1.0.0` configs unless a change is explicitly documented as a breaking migration and moved to a future major version.

## First named 1.x feature: factions

Factions are planned for a version higher than `1.0.0`, likely the first suitable minor release after public launch.

Target project:

- [factions project](../projects/factions/)

Expected scope:

- config-owned faction catalogue;
- `subsystems.factions` disabled by default;
- `!faction` player command hub;
- character-local membership and standing state;
- rank or renown support;
- shared faction requirement helpers;
- editor validation and guided editing;
- one or two representative integrations, such as faction-gated shops, quests, libraries, or location services.

## Compatibility rules

For the `1.x.x` line:

- absent new subsystem blocks must default to disabled;
- existing `1.0.0` configs should keep passing validation;
- new commands should be opt-in;
- new config sections should be additive;
- setting examples, such as Harpers data, belong in presets or owner config rather than generic aliases.

## Candidate future additions

These are placeholders, not commitments:

- economy `interview` command for acquiring location jobs before they appear as a character's available work;
- deeper faction integrations after the initial faction release;
- richer quest/faction reward flows;
- additional starter config presets;
- post-public editor convenience features;
- broader Forgotten Realms regions beyond the Sword Coast starter.
